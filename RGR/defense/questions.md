## Вопросы для защиты РГР

### 1. Какие структуры данных использованы для представления графа?
**Ответ:** Граф может храниться в двух формах: матрица смежности (`GraphMatrixForm`) и список смежности (`GraphListForm`). В обеих формах поддерживаются кратные рёбра (мультиграф). В матричной форме используется `List<List<List<E>>>`, где для каждой пары вершин хранится список рёбер. В списковой — массив `LinkedList<Node>`, каждый элемент списка содержит ребро и номер целевой вершины. Это позволяет эффективно выполнять операции добавления/удаления и обхода.

### 2. Как реализована шаблонность (generics) в классах?
**Ответ:** Класс `Graph` параметризован типами: `V extends Vertex<K, D>` (вершина), `K` (тип имени), `D` (тип данных вершины), `W` (тип веса ребра), `ED` (тип данных ребра). Классы `Vertex` и `Edge` также являются шаблонными. Это позволяет использовать любые типы для имён, весов и пользовательских данных, сохраняя типобезопасность.

### 3. Что такое дескриптор вершины и ребра? Как они связаны с идентификатором (id)?
**Ответ:** Дескриптор — это объект, который однозначно идентифицирует вершину или ребро. В нашей реализации это ссылка на объект `Vertex` или `Edge`. В программе введён уникальный целочисленный `id`, который присваивается каждому объекту при создании и никогда не меняется. Индекс вершины (позиция в списке) может изменяться при удалении других вершин, но `id` остаётся постоянным. Дескриптор позволяет быстро получить доступ к объекту, а `id` используется для операций удаления и отображения в подсказках.

### 4. Как работают итераторы и как они обрабатывают удаление текущего элемента?
**Ответ:** В классе `Graph` определены внутренние классы `VertexIterator`, `EdgeIterator` и `OutgoingEdgeIterator`. Все они реализуют интерфейс `Iterator`. При удалении текущего элемента через метод `remove()` итератор корректно сдвигается на следующий элемент, что соответствует требованию РГР. Для этого в итераторе хранится информация о последнем возвращённом элементе, и при вызове `remove()` удаляется именно он, а курсор при необходимости уменьшается.

### 5. Что такое эйлеров цикл и как он реализован в задании 2 (вариант 16)?
**Ответ:** Эйлеров цикл — это цикл, проходящий по каждому ребру графа ровно один раз. Для неориентированного графа условие существования: все вершины имеют чётную степень и граф связен. В задаче реализован двухпроходный алгоритм (алгоритм Хирхольцера): сначала строится произвольный цикл, затем он расширяется за счёт добавления неиспользованных рёбер. Алгоритм работает за O(E). Результат возвращается в виде списка рёбер в порядке обхода. Если граф не эйлеров, метод `isValid()` возвращает `false`.

### 6. Как определяется радиус и центр взвешенного орграфа (задание 3, вариант 13)?
**Ответ:** Радиус — это минимальный эксцентриситет среди всех вершин. Эксцентриситет вершины — максимальное расстояние от данной вершины до любой другой достижимой вершины. Для нахождения кратчайших путей из каждой вершины используется алгоритм Дейкстры (так как граф ориентированный и веса положительные). Затем для каждой вершины вычисляется эксцентриситет, и вершина с наименьшим эксцентриситетом объявляется центром. Радиус — значение этого эксцентриситета. Метод `result()` возвращает центр и соответствующий путь.

### 7. Какие формы представления графа поддерживаются и как переключаются?
**Ответ:** Поддерживаются L-граф (список смежности) и M-граф (матрица смежности). Переключение выполняется методами `ToListGraph()` и `ToMatrixGraph()`. Они создают новый объект соответствующей формы и копируют в него все вершины и рёбра из текущей структуры. При этом сам граф (вершины, рёбра) остаётся неизменным, меняется только способ хранения связей. Это позволяет выбирать оптимальное представление в зависимости от плотности графа.

### 8. Что такое коэффициент насыщенности графа?
**Ответ:** Коэффициент насыщенности — это отношение реального числа рёбер к максимально возможному для данного количества вершин и ориентированности. Для неориентированного графа максимальное число рёбер равно V·(V-1)/2, для ориентированного — V·(V-1). Метод `K()` возвращает это значение как `double`. Если граф не имеет вершин (V=0), возвращается -1.

### 9. Как реализована визуализация графа в GUI?
**Ответ:** Используется `GraphPanel`, наследник `JPanel`, который отрисовывает граф с помощью Java2D. Поддерживаются панорамирование (перетаскивание левой кнопкой мыши) и масштабирование (колесико мыши). Позиции вершин генерируются случайно без пересечений. При наведении курсора на вершину или ребро они подсвечиваются (вершина — оранжевым, ребро — красным толстым), курсор меняется на руку. При клике открывается диалоговое окно с информацией об объекте: для вершины — id, индекс, имя, данные; для ребра — id, начальная и конечная вершины, вес. Петли рисуются как овалы рядом с вершиной, кратные рёбра смещаются перпендикулярно направлению.

### 10. Как в программе реализованы уникальные идентификаторы (id) и удаление по id?
**Ответ:** В классах `Vertex` и `Edge` добавлено статическое поле `nextId`, которое увеличивается при каждом создании объекта. Поле `id` присваивается в конструкторе и никогда не изменяется. ID не переиспользуются даже после удаления объектов. В панели управления добавлены текстовое поле для ввода ID и кнопки «Удалить вершину по ID» и «Удалить ребро по ID». При нажатии происходит поиск объекта с таким ID в коллекциях и его удаление. Индексы вершин (позиция в списке) не связаны с ID и могут меняться при удалении, но ID остаётся уникальным идентификатором.

### 11. Что показывает всплывающая подсказка при клике на вершину и на ребро?
**Ответ:** При клике на вершину отображается:
- `id` вершины
- Индекс (позиция в списке, начиная с 0)
- Имя вершины
- Данные вершины (если заданы, иначе не выводятся)

При клике на ребро отображается:
- `id` ребра
- Начальная и конечная вершины (их имена)
- Вес ребра (если задан, иначе не выводится)

Для петель также выводится информация о петле.

### 12. Какие методы класса Graph являются основными для задач?
**Ответ:** Основные методы, используемые в задачах:
- `V()` — число вершин
- `E()` — число рёбер
- `getVertex(int index)` — получение вершины по индексу
- `getEdges(int i, int j)` — получение списка рёбер между вершинами
- `hasEdge(int i, int j)` — проверка наличия ребра
- `getEdge(...)` — получение одного ребра
- `isDirected()` — тип графа
- Итераторы: `OutgoingEdgeIterator` для обхода исходящих рёбер вершины

Эти методы предоставляют достаточную абстракцию для реализации сложных алгоритмов (Дейкстра, эйлеров цикл) без прямого доступа к внутренним структурам.

### 13. Как обрабатываются кратные рёбра и петли?
**Ответ:** Обе формы представления (`GraphListForm` и `GraphMatrixForm`) хранят список рёбер для каждой пары вершин, что позволяет иметь несколько рёбер между одними и теми же вершинами. Петли (`v1 == v2`) также разрешены. При отрисовке кратные рёбра смещаются перпендикулярно линии на равные расстояния, чтобы они не накладывались. Петли рисуются как овалы рядом с вершиной, с собственной областью клика и стрелкой для ориентированных графов. Удаление ребра по умолчанию удаляет только первое ребро между парой (можно расширить для удаления конкретного).

### 15. Как реализована генерация случайного графа с учётом параметров (вершин, рёбер, взвешенности)?
**Ответ:** Конструктор `Graph(vertexCount, edgeCount, directed, dense)` генерирует случайные рёбра. Для взвешенного графа после создания всем рёбрам назначаются случайные веса от 1 до 9 (вызывается отдельный метод). Петли добавляются с вероятностью 5%, чтобы избежать их большого количества. Количество рёбер автоматически ограничивается максимальным для выбранного типа (если пользователь запросил больше). В GUI кнопка «Случайный» использует значения из полей ввода и состояние чекбоксов (ориентированность, взвешенность), генерирует граф с этими параметрами, но форма представления выбирается случайно. При этом позиции вершин на панели не сбрасываются — остаются старые, если не нажать кнопку перетасовки.

### 16. Как в программе реализовано удаление вершины и связанных с ней рёбер?
**Ответ:** Метод `deleteVertex(int index)` или `deleteVertex(Vertex v)` удаляет вершину и все инцидентные ей рёбра. Для этого в форме представления вызывается метод `deleteEdgesFromVertex(index, directed)`, который удаляет как исходящие, так и входящие рёбра. Для ориентированного графа удаление симметричных рёбер не выполняется, для неориентированного — удаляются оба направления. После удаления рёбер обновляется счётчик `edgeCounter`. Индексы оставшихся вершин в списке сдвигаются, но их идентификаторы (id) остаются неизменными. Позиции вершин на панели не пересчитываются — они остаются прежними, даже если вершина была удалена из середины списка (графическое представление не переупорядочивается автоматически, положение каждой вершины сохраняется).
