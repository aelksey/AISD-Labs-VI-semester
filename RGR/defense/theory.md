# Теория по графам для защиты работы

## 1. Основные определения

**Граф** — это пара множеств \( G = (V, E) \), где \( V \) — непустое множество вершин (узлов), \( E \) — множество рёбер, каждое из которых соединяет две вершины (возможно, одинаковые). Рёбра могут быть ориентированными и неориентированными.

- **Неориентированный граф** — рёбра не имеют направления; ребро \(\{u, v\}\) эквивалентно \(\{v, u\}\).
- **Ориентированный граф (орграф)** — каждое ребро имеет направление; дуга \((u, v)\) отличается от \((v, u)\).
- **Смешанный граф** — содержит оба типа рёбер (в работе не используется).

**Степень вершины** — количество инцидентных ей рёбер. В орграфе различают полустепень исхода (исходящие дуги) и полустепень захода (входящие дуги). Обозначаются \(\deg^+(v)\) и \(\deg^-(v)\).

**Путь** — последовательность вершин, в которой каждые две соседние соединены ребром. **Простой путь** не содержит повторяющихся вершин. **Цикл** — замкнутый путь (начальная и конечная вершины совпадают) с отсутствием повторений среди остальных вершин. **Простой цикл** не имеет повторяющихся вершин, кроме первой/последней.

**Связность**: неориентированный граф **связен**, если между любыми двумя вершинами существует путь. Орграф **сильно связен**, если для любой пары вершин \(u, v\) существуют пути \(u \to v\) и \(v \to u\). **Компонента связности** — максимальное по включению связное подмножество вершин.

## 2. Способы представления графа

### 2.1. Матрица смежности (M-граф)

Для графа с \(n\) вершинами строится квадратная матрица \(A\) размера \(n \times n\), где:
- \(A[i][j] = 1\) (или вес ребра), если существует ребро \(i \to j\);
- \(A[i][j] = 0\) (или \(\infty\) для отсутствия ребра) в противном случае.

**Достоинства**:
- Проверка наличия ребра \(O(1)\).
- Простота реализации.

**Недостатки**:
- Память \(O(n^2)\) — неэффективна для разреженных графов.
- Добавление/удаление вершины требует \(O(n)\) изменений.

### 2.2. Список смежности (L-граф)

Для каждой вершины хранится список смежных с ней вершин (или рёбер). Для орграфа хранятся только исходящие дуги.

**Достоинства**:
- Память \(O(n + m)\) — эффективно для разреженных графов.
- Легко добавлять/удалять вершины и рёбра.

**Недостатки**:
- Проверка наличия ребра требует \(O(\deg(v))\) времени.

В работе поддерживаются обе формы с возможностью преобразования. **Коэффициент насыщенности** \(K = \frac{|E|}{|E_{\max}|}\), где \(|E_{\max}| = n(n-1)\) для орграфа и \(n(n-1)/2\) для неориентированного графа. Характеризует плотность графа.

## 3. Обходы графа

### 3.1. Поиск в глубину (Depth-First Search, DFS)

Алгоритм рекурсивно обходит граф: посещает вершину, помечает её, затем рекурсивно обходит всех непосещённых соседей. Использует стек (явно или через рекурсию).

**Свойства**:
- Позволяет обнаружить компоненты связности, циклы, точки сочленения.
- Временная сложность \(O(n + m)\).
- Для орграфа может использоваться для топологической сортировки (ациклические графы).

**Псевдокод**:
```
DFS(v):
    visited[v] = true
    for each u in neighbours(v):
        if not visited[u]:
            DFS(u)
```

### 3.2. Поиск в ширину (Breadth-First Search, BFS)

Использует очередь: начинаем с начальной вершины, затем посещаем всех её соседей, затем соседей соседей и т.д.

**Свойства**:
- Находит кратчайшие пути в невзвешенном графе (по числу рёбер).
- Сложность \(O(n + m)\).

**Применение**: проверка связности, поиск кратчайших путей, алгоритмы для двудольных графов.

## 4. Эйлеровы циклы (задание 2, вариант 16)

**Эйлеров цикл** — замкнутый маршрут, проходящий по каждому ребру графа ровно один раз. Назван в честь Леонарда Эйлера, решившего задачу о семи кёнигсбергских мостах (1736 г.). Граф, содержащий эйлеров цикл, называется **эйлеровым**.

**Теорема Эйлера** (для неориентированного графа):
Связный неориентированный граф содержит эйлеров цикл **тогда и только тогда**, когда степени всех вершин чётные.

*Доказательство идеи*: Необходимость очевидна: при каждом входе в вершину мы должны из неё выйти. Достаточность доказывается конструктивно (алгоритмом).

**Алгоритм построения эйлерова цикла** (алгоритм Хирхольцера, двухпроходный):
1. Начинаем с произвольной вершины и строим произвольный цикл, следуя рёбрам и удаляя их.
2. Если остались неиспользованные рёбра, находим в построенном цикле вершину, у которой есть такие рёбра.
3. Строим из неё новый цикл и вставляем его в основной.
4. Повторяем, пока все рёбра не будут исчерпаны.

**Сложность**: \(O(m)\).

**Применение**:
- Задача китайского почтальона (поиск маршрута с минимальным повторным прохождением рёбер).
- Диагностика электрических цепей.
- Сборка генома из фрагментов (де Брёйна).

В работе реализован двухпроходный построитель эйлерова цикла для неориентированных графов (сначала проверяется чётность степеней и связность). Результат возвращается в виде списка рёбер в порядке обхода.

## 5. Кратчайшие пути и радиус графа (задание 3, вариант 13)

### 5.1. Алгоритм Дейкстры

Решает задачу о кратчайших путях из одной вершины во взвешенном графе с **неотрицательными** весами рёбер.

**Идея**:
- Поддерживаем множество посещённых вершин с известными расстояниями.
- Инициализируем расстояния: для начальной вершины 0, для остальных — \(\infty\).
- На каждом шаге выбираем непосещённую вершину с минимальным расстоянием и помечаем её как посещённую.
- Для всех её соседей выполняем **релаксацию**:
  если \(dist[u] + w(u, v) < dist[v]\), то обновляем \(dist[v]\).

**Реализация с двоичной кучей** даёт сложность \(O(m \log n)\). В работе для простоты и малых \(n\) используется квадратичная версия \(O(n^2)\).

**Доказательство корректности**: основано на жадном выборе вершины с минимальным расстоянием; для неотрицательных весов любой более короткий путь не может проходить через непосещённую вершину.

### 5.2. Эксцентриситет, радиус и центр графа

Пусть \(dist(u, v)\) — длина кратчайшего пути от \(u\) до \(v\) (или \(\infty\), если пути нет). Для орграфа расстояние может быть несимметричным.

- **Эксцентриситет** вершины \(v\): \(ecc(v) = \max_{u \in V} dist(v, u)\) (максимальное расстояние до любой достижимой вершины).
- **Радиус** графа: \(r = \min_{v \in V} ecc(v)\).
- **Диаметр** графа: \(d = \max_{v \in V} ecc(v)\).
- **Центр** графа — множество вершин, для которых \(ecc(v) = r\).

**Поиск радиуса и центра**:
1. Вычисляем кратчайшие расстояния между всеми парами вершин. Для взвешенного орграфа с положительными весами используем \(n\) раз алгоритм Дейкстры (или алгоритм Флойда-Уоршелла).
2. Для каждой вершины \(v\) находим максимальное расстояние до других вершин.
3. Находим минимум среди этих максимумов — радиус; соответствующие вершины — центр.

В работе результат возвращается в виде индекса вершины-центра и значения радиуса, а также путь от центра до вершины, на которой достигается максимальное расстояние (демонстрационный путь).

## 6. Ориентированные графы

**Свойства**:
- Полустепень захода \(\deg^-(v)\) и полустепень исхода \(\deg^+(v)\).
- Сильная связность: для любой пары вершин существуют пути в обе стороны.
- **Топологическая сортировка** возможна только для ациклических орграфов (DAG — Directed Acyclic Graph). Используется в планировании задач.

**Алгоритм Косарайю для поиска сильно связных компонент**:
1. Выполнить DFS на исходном графе, записывая порядок завершения вершин (стек).
2. Построить транспонированный граф (обратить все дуги).
3. Выполнять DFS в порядке из стека; каждый вызов DFS даёт одну компоненту.

## 7. Взвешенные графы

**Вес ребра** — числовая характеристика, интерпретируемая как длина, стоимость, время, пропускная способность.

**Основные задачи**:
- Кратчайшие пути: Дейкстра (неотрицательные веса), Беллман-Форд (допускает отрицательные, но без отрицательных циклов), Флойд-Уоршелл (все пары).
- Минимальное остовное дерево: алгоритмы Прима и Крускала.
- Максимальный поток: Форд-Фалкерсон, Эдмондс-Карп.

В работе веса целые положительные (от 1 до 9 при генерации), можно редактировать через GUI. Алгоритм Дейкстры полностью применим.

## 8. Мультиграфы и петли

**Мультиграф** — граф, в котором допускаются **кратные рёбра** (несколько рёбер между одной парой вершин). **Петля** — ребро, соединяющее вершину с самой собой.

**Особенности**:
- Степень вершины в мультиграфе — суммарное количество инцидентных рёбер (с учётом кратности). Петля добавляет 2 к степени.
- Эйлеров цикл в мультиграфе существует при тех же условиях (все вершины имеют чётную степень), но петли и кратные рёбра не мешают, а наоборот, облегчают построение.
- В матричном представлении обычно хранят число кратных рёбер; в списковом — несколько записей.

В работе обе формы представления хранят для каждой пары вершин **список рёбер**, что естественным образом поддерживает мультиграф и петли. При отрисовке кратные рёбра смещаются перпендикулярно направлению, чтобы быть видимыми, а петли рисуются как овалы рядом с вершиной.

## 9. Проектирование АТД «Простой граф» (согласно заданию)

В работе реализован шаблонный класс `Graph` со следующей архитектурой:

- **Абстрактный класс `GraphForm`** — определяет интерфейс для работы с конкретной формой представления.
- **Конкретные классы `GraphMatrixForm` и `GraphListForm`** — реализуют матричный и списочный способы хранения.
- **Дескрипторы вершин и рёбер** — классы `Vertex` и `Edge` содержат поля: имя, данные, вес, а также уникальный целочисленный `id`. `id` присваивается при создании и никогда не изменяется; это облегчает удаление по идентификатору.
- **Итераторы** — внутренние классы `VertexIterator`, `EdgeIterator`, `OutgoingEdgeIterator`, реализующие стандартный паттерн итератора. Особенность: при удалении текущего элемента через `remove()` итератор корректно сдвигается на следующий, что соответствует требованию РГР.
- **Вспомогательные методы**: преобразование L↔M, вычисление коэффициента насыщенности, добавление/удаление вершин и рёбер, проверка существования ребра, получение числа вершин и рёбер и т.д.

## 10. Теоретическая сложность реализованных операций и алгоритмов

| Операция / Алгоритм | Форма представления | Сложность |
|---------------------|---------------------|-----------|
| Проверка ребра      | L-граф              | \(O(\deg(v_1))\) |
| Проверка ребра      | M-граф              | \(O(1)\) |
| Добавление/удаление вершины | любая | \(O(n)\) (из-за сдвига индексов) |
| Добавление/удаление ребра | L-граф | \(O(1)\) (для добавления), \(O(\deg(v_1))\) для удаления |
| Добавление/удаление ребра | M-граф | \(O(1)\) |
| Поиск в глубину (DFS) | любая | \(O(n + m)\) |
| Эйлеров цикл (алгоритм Хирхольцера) | любая | \(O(m)\) |
| Алгоритм Дейкстры (без кучи) | любая | \(O(n^2)\) |
| Алгоритм Дейкстры (с двоичной кучей) | любая | \(O(m \log n)\) |
| Вычисление радиуса (n запусков Дейкстры) | любая | \(O(n^3)\) для плотных графов |
| Преобразование L↔M | — | \(O(n^2)\) |

В работе из-за визуального ограничения \(V < 20\) сложность не критична.
