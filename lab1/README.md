# Отчет по лабораторной работе №1
## «Коллекция данных - список»

**Дисциплина:** Алгоритмы и структуры данных
**Вариант:** 6 (Кольцевой, односвязный список на базе адресных указателей)
**Язык программирования:** C++

---

### 1. Цель работы

Освоение технологии реализации позиционных, линейных коллекций на примере АТД «Список». Целью являлось проектирование, реализация и тестирование шаблонного класса «Список» с кольцевой односвязной структурой хранения на базе адресных указателей.

### 2. Общее задание и вариант задания

**Общее задание:**
Спроектировать, реализовать и провести тестовые испытания АТД «Список» для коллекции, содержащей данные произвольного типа. Тип данных задаётся клиентской программой. Интерфейс АТД должен включать стандартные операции: конструкторы, деструктор, опрос размера, очистка, проверка на пустоту, поиск, чтение/изменение по позиции, вставка, удаление, а также прямой итератор.

**Вариант задания (№6):**
Структура данных - кольцевая, односвязная, на базе адресных указателей.
Прямой итератор должен обеспечивать обход элементов от головы к хвосту. Так как список кольцевой, итератор `end()` должен сигнализировать о завершении обхода после прохождения всех элементов (т.е. после `size()` шагов).

### 3. Формат АТД «Список»

**Общая характеристика:**
Список представляет позиционно-ориентированную, линейную последовательность с доступом к элементам по номеру позиции или по значению. В данной реализации используется кольцевая односвязная структура: последний элемент списка хранит указатель не на `nullptr`, а на первый элемент (голову). Доступ к элементам возможен только последовательно, начиная с головы.

**Данные:**
*   **Параметры:**
    *   `T` - тип элементов, хранящихся в списке.
    *   `size` – количество элементов в списке.
*   **Структура хранения коллекции:**
    *   Узлы (`Node`), размещенные в динамической памяти.
    *   Каждый узел содержит поле данных `data` типа `T` и указатель `next` на следующий узел.
    *   Указатель `head` на первый узел списка.

**Операции:**

| Операция | Вход | Предусловия | Процесс | Выход/Постусловия |
| :--- | :--- | :--- | :--- | :--- |
| `Конструктор` | нет | нет | Создание пустого списка | Создан пустой список с `head = nullptr`, `size = 0` |
| `Конструктор копирования` | `other` - копируемый список | нет | Создание копии списка `other` | Создана глубокая копия всех узлов списка `other` |
| `Деструктор` | нет | нет | Очистка динамической памяти | Все узлы удалены, память освобождена |
| `size()` | нет | нет | Возврат текущего количества элементов | Значение `size` |
| `empty()` | нет | нет | Проверка, пуст ли список | `true` - если `size == 0`, иначе `false` |
| `clear()` | нет | нет | Удаление всех элементов списка | `size = 0`, `head = nullptr` |
| `contains(value)` | `value` | нет | Проверка наличия значения | `true` - если значение найдено, иначе `false` |
| `at(pos)` | `pos` - позиция | `0 ≤ pos < size` | Доступ по чтению/записи к элементу | Ссылка на данные узла по позиции `pos` |
| `set(pos, value)` | `pos`, `value` | `0 ≤ pos < size` | Изменение значения | Данные узла по позиции `pos` заменены на `value` |
| `indexOf(value)` | `value` | нет | Поиск позиции значения | Позиция первого вхождения или -1, если не найдено |
| `push_back(value)` | `value` | нет | Добавление в конец | Новый узел с `value` добавлен, `size++` |
| `insert(pos, value)` | `pos`, `value` | `0 ≤ pos ≤ size` | Вставка в позицию | Новый узел вставлен, `size++` |
| `remove_value(value)` | `value` | нет | Удаление по значению | Удалено первое вхождение, `size--`; `true` - если удалено |
| `remove_at(pos)` | `pos` | `0 ≤ pos < size` | Удаление по позиции | Удален элемент, `size--`; `true` - если удалено |

### 4. Формат АТД «Прямой итератор списка»

**Общая характеристика:**
Итератор предоставляет клиентской программе механизм последовательного доступа к элементам списка для чтения и записи. Итератор может находиться в состоянии «установлен» (указывает на элемент) или «не установлен» (`end()`).

**Параметры:**
*   `current` - указатель на текущий узел списка.
*   `head` - указатель на голову списка (для контроля завершения обхода).
*   `steps` - количество пройденных шагов.
*   `listSize` - размер списка на момент создания итератора.

**Операции:**

| Операция | Вход | Предусловия | Процесс | Выход/Постусловия |
| :--- | :--- | :--- | :--- | :--- |
| `operator*` | нет | Итератор установлен | Получение ссылки на данные текущего узла | Ссылка на `current->data` |
| `operator++` (преф.) | нет | Итератор установлен | Перемещение на следующий элемент | Итератор указывает на следующий узел. Если пройдено `size` шагов, итератор становится равным `end()`. |
| `operator==` | `other` итератор | нет | Проверка на равенство | `true`, если оба итератора указывают на один и тот же узел |
| `operator!=` | `other` итератор | нет | Проверка на неравенство | `true`, если итераторы указывают на разные узлы |

### 5. Справочное определение класса на C++

*(Приведено в файле `include/circular_list.h`. Основные публичные методы перечислены ниже)*

```cpp
template <typename T>
class CircularList {
public:
    // --- Конструкторы и деструктор ---
    CircularList();
    CircularList(const CircularList<T>& other);
    ~CircularList();

    // --- Основные операции ---
    int size() const;
    bool empty() const;
    void clear();
    bool contains(const T& value) const;
    T& at(int pos);
    const T& at(int pos) const;
    void set(int pos, const T& value);
    int indexOf(const T& value) const;
    void push_back(const T& value);
    void insert(int pos, const T& value);
    bool remove_value(const T& value);
    bool remove_at(int pos);

    // --- Итератор ---
    class Iterator;
    Iterator begin();
    Iterator end();

    // --- Для отладки ---
    void print() const;
};
```
