# 🌳 2-3 Дерево: Визуализация и Анализ Сложности

## 📋 Описание проекта

Проект представляет собой комплексную реализацию 2-3 дерева с графическим интерфейсом для визуализации и инструментами для анализа временной сложности операций. 2-3 дерево - это сбалансированное дерево поиска, в котором каждый внутренний узел может иметь 2 или 3 дочерних элемента.

## ✨ Основные возможности

### 1. Визуализация дерева
- 📊 Визуальное отображение структуры 2-3 дерева
- 🟢 Зеленые узлы - внутренние узлы (отображают ключи)
- 🔵 Синие узлы - листья (отображают значения)
- 🖱️ Панорамирование: зажать левую кнопку мыши и перетаскивать
- 🔍 Масштабирование: колесико мыши
- ⌨️ Горячие клавиши: R - сброс вида, + / - - масштабирование

### 2. Управление деревом
- ➕ Добавление элементов
- 🗑️ Удаление элементов
- 🎲 Генерация случайного дерева заданного размера
- 📊 Генерация вырожденного дерева (отсортированные значения)

### 3. Настройка отображения
- 📏 Горизонтальное расстояние между узлами (30-300)
- 📐 Вертикальное расстояние между уровнями (30-250)
- ⚪ Радиус узлов (15-60)
- 🔤 Размер текста в узлах (8-24)

### 4. Анализ сложности
- 📈 Измерение времени выполнения операций Insert, Search, Delete
- 📊 Сравнение производительности сбалансированного и вырожденного деревьев
- 📉 Визуализация результатов в графиках
- 💾 Экспорт результатов в CSV формат

## 🛠️ Технологии

- **Язык**: Java (Swing для GUI)
- **Сборка**: JDK 8 или выше
- **Визуализация**: Python 3 (pandas, matplotlib) для построения графиков

- Time (nanoseconds/microseconds/milliseconds)

## 📊 Интерпретация результатов

### Сбалансированное (случайное) дерево
- **Insert**: O(log n) - логарифмическая сложность
- **Search**: O(log n) - очень быстрый поиск
- **Delete**: O(log n) - эффективное удаление

### Вырожденное дерево
- **Insert**: O(n) - линейная сложность
- **Search**: O(n) - медленный поиск
- **Delete**: O(n) - неэффективное удаление

### Пример производительности (для 100,000 элементов)

| Операция | Случайное дерево | Вырожденное дерево | Разница |
|----------|-----------------|-------------------|---------|
| Insert   | 0.195 мс        | 12,500 мс         | 64,000x |
| Search   | 0.045 мс        | 6,250 мс          | 138,000x |
| Delete   | 0.285 мс        | 18,750 мс         | 65,000x |

## 🎯 Теоретическая сложность

### 2-3 Дерево (сбалансированное)
- **Высота дерева**: O(log n)
- **Поиск**: O(log n) сравнений
- **Вставка**: O(log n) с перебалансировкой
- **Удаление**: O(log n) с перебалансировкой

### Вырожденный случай (худший)
- **Высота дерева**: O(n)
- **Поиск**: O(n) сравнений
- **Вставка**: O(n) с перебалансировкой
- **Удаление**: O(n) с перебалансировкой

**Решение**: Убедитесь, что все классы скомпилированы
```bash
javac com/lab/*.java
```

### Проблема: Пустой CSV файл
**Решение**: Запустите анализатор с меньшими размерами
```java
// Измените в Tree23ComplexityAnalyzer.java:
int[] sizes = {1000, 5000, 10000}; // вместо больших значений
```

### Проблема: Не отображается текст в полях ввода
**Решение**: Текст должен быть черным на белом фоне. Если нет, проверьте:
- Цвет фона: Color.WHITE
- Цвет текста: Color.BLACK
- Цвет курсора: Color.BLACK
