# 📚 Теория для защиты лабораторной работы: 2-3 Дерево

## 1. Основные понятия и определения

### 1.1 Что такое 2-3 дерево?

**2-3 дерево** - это сбалансированное дерево поиска, в котором каждый внутренний узел может иметь 2 или 3 дочерних элемента. Оно является частным случаем B-дерева порядка 3.

### 1.2 Свойства 2-3 дерева

1. **Свойство узлов**:
   - 2-узел: содержит 1 ключ и имеет 2 дочерних элемента
   - 3-узел: содержит 2 ключа и имеет 3 дочерних элемента

2. **Свойство упорядоченности** (для узла с ключами k1 < k2):
   - Левое поддерево: ключи < k1
   - Среднее поддерево: k1 < ключи < k2
   - Правое поддерево: ключи > k2

3. **Свойство сбалансированности**:
   - Все листья находятся на одном уровне
   - Длина пути от корня до любого листа одинакова

### 1.3 Структура узла

```
2-узел:         3-узел:
   [k1]           [k1, k2]
  /   \          /  |  \
 A     B        A   B   C
где A < k1 < B   где A < k1 < B < k2 < C
```

## 2. Операции над 2-3 деревом

### 2.1 Поиск (Search)

**Алгоритм**:
1. Начинаем с корня
2. Сравниваем искомый ключ с ключами в текущем узле
3. Переходим в соответствующее поддерево
4. Повторяем до нахождения ключа или достижения листа

**Сложность**: O(log n) в сбалансированном случае

**Псевдокод**:
```
function search(node, key):
    if node is leaf:
        return node.value if node.key == key else null
    
    if key < node.key1:
        return search(node.left, key)
    else if node is 2-node or key < node.key2:
        return search(node.middle, key)
    else:
        return search(node.right, key)
```

### 2.2 Вставка (Insert)

**Алгоритм**:
1. Находим лист, куда нужно вставить элемент
2. Если в листе есть место (1 ключ), просто вставляем
3. Если лист полный (2 ключа):
   - Разбиваем лист на два
   - Поднимаем средний ключ в родителя
   - Если родитель становится полным - рекурсивно разбиваем

**Случаи вставки**:

**Случай 1**: Вставка в 2-узел
```
    [5]              [3,5]
   /   \     →       / | \
 [3]   [7]         [2][4][7]
```

**Случай 2**: Вставка в 3-узел (с разбиением)
```
      [5]                    [5,9]
     /   \                  /  |  \
   [3,7] [11]     →       [3] [7] [11]
   / | \                 / \ / \ / \
```

**Сложность**: O(log n) амортизированная

### 2.3 Удаление (Delete)

**Алгоритм**:
1. Находим удаляемый элемент
2. Если элемент в листе:
   - Просто удаляем, если после удаления в узле есть ключ
   - Если узел становится пустым - выполняем перебалансировку
3. Если элемент во внутреннем узле:
   - Находим преемника (следующий по величине элемент)
   - Заменяем удаляемый элемент преемником
   - Удаляем преемник из листа

**Случаи перебалансировки**:
1. **Перераспределение**: брат может отдать один ключ
2. **Слияние**: объединение с братом, если он также имеет 1 ключ

**Сложность**: O(log n) амортизированная

## 3. Анализ сложности

### 3.1 Теоретическая сложность операций

| Операция | Сбалансированное дерево | Вырожденное дерево |
|----------|------------------------|-------------------|
| Search   | O(log n)               | O(n)              |
| Insert   | O(log n)               | O(n)              |
| Delete   | O(log n)               | O(n)              |

### 3.2 Доказательство высоты дерева

**Теорема**: Высота 2-3 дерева с n ключами не превышает log₂(n+1)

**Доказательство**:
- Минимальное количество узлов на глубине h: 2^h
- Максимальное количество узлов на глубине h: 3^h
- Все листья на глубине h, значит n ≥ 2^h
- Следовательно, h ≤ log₂(n)

### 3.3 Сравнение с другими структурами

| Структура | Search | Insert | Delete | Память |
|-----------|--------|--------|--------|---------|
| 2-3 Tree  | O(log n) | O(log n) | O(log n) | O(n) |
| BST (avg) | O(log n) | O(log n) | O(log n) | O(n) |
| BST (worst)| O(n)   | O(n)    | O(n)    | O(n) |
| AVL Tree  | O(log n) | O(log n) | O(log n) | O(n) |
| Hash Table| O(1)   | O(1)    | O(1)    | O(n) |

## 5. Инварианты и проверка корректности

### 5.1 Основные инварианты

1. **Инвариант упорядоченности**:
   - Для каждого узла ключи упорядочены по возрастанию
   - Все ключи в левом поддереве меньше первого ключа
   - Все ключи в правом поддереве больше последнего ключа

2. **Инвариант сбалансированности**:
   - Все листья находятся на одной глубине
   - Количество детей у узла равно количеству ключей + 1

3. **Инвариант связности**:
   - Листья связаны в двусвязный список
   - prev указывает на предыдущий лист
   - next указывает на следующий лист

### 5.2 Проверка инвариантов

```java
public boolean invariant() {
    int[] depths = depths();           // Проверка высоты
    boolean linkedListCorrect = leafListInvariant(); // Проверка списка
    return depths[0] == depths[1] && linkedListCorrect;
}
```

## 6. Экспериментальный анализ

### 6.1 Методика тестирования

1. **Параметры тестирования**:
   - Размеры: 1000, 5000, 10000, 50000, 100000
   - Количество итераций: 50-100 для усреднения
   - Измерение времени в наносекундах

2. **Типы тестов**:
   - Случайные данные (сбалансированное дерево)
   - Отсортированные данные (вырожденное дерево)

### 6.2 Ожидаемые результаты

**Случайное дерево (O(log n))**:
- Время растет медленно с увеличением n
- Для n=1000 → 100000 время увеличивается ~2-3 раза

**Вырожденное дерево (O(n))**:
- Время растет линейно с увеличением n
- Для n=1000 → 100000 время увеличивается ~100 раз

### 6.3 Формулы для расчета

```java
// Ожидаемое время для случайного дерева
T_random(n) = c * log₂(n)

// Ожидаемое время для вырожденного дерева
T_degenerate(n) = c * n

// Коэффициент ускорения
Speedup = T_degenerate(n) / T_random(n)
```

## 7. Преимущества и недостатки

### 7.1 Преимущества

1. **Самобалансировка**: Не требует дополнительных операций балансировки
2. **Гарантированная сложность**: O(log n) в худшем случае
3. **Эффективность**: Хорошо работает с дисковыми операциями
4. **Простота**: Проще реализации AVL и красно-черных деревьев

### 7.2 Недостатки

1. **Память**: Больше служебной информации
2. **Сложность реализации**: Сложнее обычного BST
3. **Не для кэша**: Плохая локальность данных
4. **Перебалансировка**: Может быть затратной при частых операциях

## 8. Применение на практике

### 8.1 Реальные применения

1. **Файловые системы**: B-деревья (обобщение 2-3 деревьев)
2. **Базы данных**: Индексы в СУБД
3. **Компиляторы**: Символьные таблицы
4. **Маршрутизация**: Таблицы маршрутизации

### 8.2 Когда использовать

**Использовать 2-3 дерево, когда**:
- Нужна гарантированная производительность O(log n)
- Данные поступают в произвольном порядке
- Частота операций поиска/вставки/удаления высока

**Не использовать, когда**:
- Данные можно хранить в массиве с бинарным поиском
- Требуется простая реализация
- Важна экономия памяти

### 10.2 Подсчет высоты дерева

```java
private int height(int node) {
    if (isLeaf(node)) return 1;
    int maxHeight = 0;
    for (int i = 0; i < treeNodes[node].cCount; i++) {
        maxHeight = Math.max(maxHeight, 
            height(treeNodes[node].children[i].child));
    }
    return maxHeight + 1;
}
```

### 11.1 Ошибки при реализации
