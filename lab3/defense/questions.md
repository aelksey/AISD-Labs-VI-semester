В 2-3 дереве **расщепления (split)** при удалении не бывает. Расщепление происходит только при **добавлении** элементов, когда временно появляется 4-й ключ (в узле 4→2+2).

При **удалении** возможны обратные операции: **перераспределение (transfer)** и **слияние (merge)**. Вершина «переезжает» в другую ветку именно при **перераспределении**.

## Когда вершина «переезжает» при удалении
Если после удаления ключа в узле остаётся меньше **1 ключа** (для внутреннего) или **меньше 1 ключа** для листа (т.е. 0), то узел становится **недоукомплектованным (underflow)**. Тогда:

1. Смотрим на **соседний братский узел** (слева или справа).
2. Если у брата есть **лишний ключ** (в узле 3 ключа → может отдать 1), делаем **перераспределение**:
   - Родительский ключ спускается в проблемный узел.
   - Крайний ключ из брата поднимается в родителя.
   - Это и есть «переезд» вершины: ключ меняет своего владельца, перемещаясь между братьями через родителя.

## Пример удаления с перераспределением

Исходное 2-3 дерево:

```
       [30, 50]
      /   |    \
   [10,20] [40] [60,70]
```

Удаляем `10` из левого листа `[10,20]`:

- В листе остаётся `[20]` (1 ключ — норма).
- Всё хорошо.

А если удалить `20` из `[10,20]`:

1. Лист становится `[10]` — **норма** (1 ключ).
2. Ничего не происходит.

Чтобы **вызвать underflow**, удалим ключ из узла, где был **один ключ**. Например, удалим `40` из узла `[40]`:

```
       [30, 50]
      /   |    \
   [10,20] []   [60,70]
```
- Лист `[40]` стал пустым (0 ключей).
- Берём брата слева `[10,20]` (у него 2 ключа, может отдать).
- Родительский ключ `30` спускается в пустой лист → лист `[30]`.
- Крайний правый ключ брата `20` поднимается в родителя на место `30` → родитель `[20, 50]`.
- Брат становится `[10]`.

Дерево после перераспределения:

```
       [20, 50]
      /   |    \
    [10] [30] [60,70]
```

Ключ `20` **«переехал»** из левого листа через родителя в пустой лист (бывший `[40]`).

## Когда перераспределение невозможно
Если у брата **тоже всего 1 ключ** (оба тощие), то делают **слияние** (merge) проблемного узла с братом и опускают ключ из родителя — переезд происходит, но это не «переезд вершины», а склеивание.

**Коротко:** вершина «переезжает» в другую ветку при удалении **только при перераспределении**, когда у соседа есть лишний ключ (≥2 ключей), а у проблемного узла стал 0 ключей (после удаления из 1-ключевого узла).
