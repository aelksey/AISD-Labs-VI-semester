## 🔧 ПРАКТИЧЕСКИЕ ЗАДАНИЯ

Лучший способ закрепить теорию — решить несколько задач на вставку в хеш-таблицу вручную.

### Задание 1: Квадратичное зондирование (как в вашем варианте)

Условие: Хеш-таблица размера 7, хеш-функция `h(k) = k % 7`. Вставить ключи: `23, 13, 21, 14, 7, 8, 15`. Разрешение коллизий — квадратичное зондирование.

**Решение (для самопроверки):**
```
h0(23) = 2 → [2]
h0(13) = 6 → [6]
h0(21) = 0 → [0]
h0(14) = 0 (коллизия) → h1 = (0 + 1²) % 7 = 1 → [1]
h0(7) = 0 (коллизия) → h1 = 1 (занято) → h2 = (0 + 2²) % 7 = 4 → [4]
h0(8) = 1 (коллизия) → h1 = 2 (занято) → h(-1) = (1 - 1²) = 0 (занято) → h2 = 5 → [5]
h0(15) = 1 → h1 = 2, h(-1) = 0, h2 = 5, h(-2) = 4, h3 = 3 → [3]
```

Итоговая таблица:
| Индекс | 0 | 1 | 2 | 3 | 4 | 5 | 6 |
|--------|---|---|---|---|---|---|---|
| Ключ | 21 | 14 | 23 | 15 | 7 | 8 | 13 |

*Источник: KFUPM Lab Assignment*

### Задание 2: Хеширование через сумму цифр + квадратичное зондирование

Условие: Хеш-таблица размера 7. Хеш-функция: `hash(k) = сумма цифр ключа % 7`. Вставить: `5005, 6374, 2637, 7897, 3453, 2703, 7151`.

**Решение (для самопроверки):**
```
5005 → 5+0+0+5 = 10 → 10 % 7 = 3 → [3]
6374 → 6+3+7+4 = 20 → 20 % 7 = 6 → [6]
2637 → 2+6+3+7 = 18 → 18 % 7 = 4 → [4]
7897 → 7+8+9+7 = 31 → 31 % 7 = 3 (коллизия) → (3+1²)%7=4 (занято) → (3+2²)%7=0 → [0]
3453 → 3+4+5+3 = 15 → 15 % 7 = 1 → [1]
2703 → 2+7+0+3 = 12 → 12 % 7 = 5 → [5]
7151 → 7+1+5+1 = 14 → 14 % 7 = 0 (коллизия) → (0+1²)%7=1 (занято) → (0+2²)%7=4 (занято) → (0+3²)%7=2 → [2]
```

Итоговая таблица:
| 0 | 1 | 2 | 3 | 4 | 5 | 6 |
|---|---|---|---|---|---|---|
|7897|3453|7151|5005|2637|2703|6374|

*Источник: JHU DSA Course*

## 📖 КЛЮЧЕВЫЕ ТЕРМИНЫ ДЛЯ ЗАПОМИНАНИЯ

| Термин (рус/англ) | Что это простыми словами |
|-------------------|--------------------------|
| **Хеш-таблица / Hash Table** | Структура данных типа «ключ → значение» |
| **Хеш-функция / Hash Function** | Правило, которое превращает ключ в индекс (число) |
| **Коллизия / Collision** | Когда два разных ключа получили одинаковый индекс |
| **Коэффициент заполнения / Load Factor (α)** | α = количество элементов / размер таблицы |
| **Метод цепочек / Chaining** | В каждой ячейке — список элементов  |
| **Открытая адресация / Open Addressing** | Ищем другую свободную ячейку при коллизии |
| **Линейное зондирование / Linear Probing** | Шаг = 1: проверяем следующую ячейку |
| **Квадратичное зондирование / Quadratic Probing** | Шаг = 1², 2², 3²… (ваш вариант) |
| **Двойное хеширование / Double Hashing** | Шаг вычисляется второй хеш-функцией |
