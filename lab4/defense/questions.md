### Вопрос 1. Приведите размер хеш-таблицы с открытой адресацией, предназначенной для хранения 200 элементов, если она использует мультипликативный метод хеширования, модульный метод хеширования.

**Ответ:**

Для хеш-таблицы с открытой адресацией важно выбрать размер таблицы m, который является **простым числом**, чтобы снизить вероятность коллизий и обеспечить равномерное распределение ключей.

**Для модульного хеширования (h(k) = k mod m):**
- Рекомендуется использовать простые числа (числа Мерсенна)
- Для 200 элементов рекомендуется коэффициент заполнения α ≈ 0.5-0.7
- Размер таблицы: m = n / α
  - При α = 0.5: m = 200 / 0.5 = 400
  - При α = 0.7: m = 200 / 0.7 ≈ 286
- Ближайшие простые числа: **269, 271, 277, 281, 283, 293**
- **Рекомендуемый размер: 271 или 283** (простые числа, близкие к оптимальному значению)

**Для мультипликативного хеширования (h(k) = ⌊m × (k × A mod 1)⌋):**
- В этом методе качество хеш-функции мало зависит от m
- Рекомендуется выбирать m как **степень двойки** (для быстрого вычисления через сдвиг)
- Для 200 элементов:
  - m = 256 (2⁸) или 512 (2⁹)
- **Рекомендуемый размер: 256** (оптимальный баланс между памятью и производительностью)

### Вопрос 5. Приведите соответствующий числу 1263454756 ключ, полученный методом свёртки для хеш-таблицы размером 1000.

**Ответ:**

Метод свёртки: разбиваем число на группы цифр и складываем их.

Число: **1 263 454 756** (10 цифр)

**Шаг 1: Разбиение на группы по 3 цифры (так как размер таблицы 1000 → 3 десятичных разряда)**

```
Группа 1: 1    (или 001)
Группа 2: 263
Группа 3: 454
Группа 4: 756
```

**Шаг 2: Сложение групп (с учётом позиции - модифицированная свёртка)**

Простая свёртка (без учёта порядка):
```
k' = 001 + 263 + 454 + 756 = 1,474
```
Поскольку k' > 999, выполняем повторную свёртку:
```
1,474 → 1 + 474 = 475
```

**Модифицированная свёртка с учётом порядка (метод XOR с порядковым номером):**

```
Группа 1: 001 XOR 1 = 0
Группа 2: 263 XOR 2 = 265
Группа 3: 454 XOR 3 = 453
Группа 4: 756 XOR 4 = 760

Сумма: 0 + 265 + 453 + 760 = 1,478
Повторная свёртка: 1,478 → 1 + 478 = 479
```

**Ответ: k' = 475 (или 479 для модифицированного метода)**

### Вопрос 6. Какая последовательность ячеек получается в результате квадратичного зондирования, если h(k) = k mod 11 и k = 19?

**Ответ:**

Размер таблицы m = 11
h(k) = k mod 11 = 19 mod 11 = **8**

Формула квадратичного зондирования: **h(k, i) = (h(k) + c₁×i + c₂×i²) mod m**

Для стандартного квадратичного зондирования обычно используют:
- c₁ = 1, c₂ = 1

**Вычисление последовательности для i = 0, 1, 2, ...**

| i | h(k,i) = (8 + i + i²) mod 11 | Ячейка |
|---|------------------------------|--------|
| 0 | (8 + 0 + 0) mod 11 = 8 | T[8] |
| 1 | (8 + 1 + 1) mod 11 = 10 | T[10] |
| 2 | (8 + 2 + 4) mod 11 = 14 mod 11 = 3 | T[3] |
| 3 | (8 + 3 + 9) mod 11 = 20 mod 11 = 9 | T[9] |
| 4 | (8 + 4 + 16) mod 11 = 28 mod 11 = 6 | T[6] |
| 5 | (8 + 5 + 25) mod 11 = 38 mod 11 = 5 | T[5] |
| 6 | (8 + 6 + 36) mod 11 = 50 mod 11 = 6 | T[6] (начинает повторяться) |
| 7 | (8 + 7 + 49) mod 11 = 64 mod 11 = 9 | T[9] |

**Последовательность индексов: 8 → 10 → 3 → 9 → 6 → 5 → 6 → 9 → ...**

Примечание: При некоторых константах квадратичное зондирование может не покрыть все ячейки таблицы. Для гарантии покрытия нужно выбирать c₁, c₂ и m специальным образом (например, m - степень двойки, c₁ и c₂ - нечётные).

### Вопрос 7. Какая последовательность зондируемых ячеек получается в результате двойного хеширования: h(k) = k mod 11, h₁(k) = 1 + (k mod 10) и k = 19?

**Ответ:**

Размер таблицы m = 11

**Шаг 1: Вычисление первой хеш-функции**
```
h(k) = k mod 11 = 19 mod 11 = 8
```

**Шаг 2: Вычисление второй хеш-функции (шаг)**
```
h₁(k) = 1 + (k mod 10) = 1 + (19 mod 10) = 1 + 9 = 10
```

**Шаг 3: Формула двойного хеширования**
```
h(k, i) = (h(k) + i × h₁(k)) mod m
```

**Вычисление последовательности для i = 0, 1, 2, ...**

| i | h(k,i) = (8 + i × 10) mod 11 | Ячейка |
|---|-----------------------------|--------|
| 0 | (8 + 0) mod 11 = 8 | T[8] |
| 1 | (8 + 10) mod 11 = 18 mod 11 = 7 | T[7] |
| 2 | (8 + 20) mod 11 = 28 mod 11 = 6 | T[6] |
| 3 | (8 + 30) mod 11 = 38 mod 11 = 5 | T[5] |
| 4 | (8 + 40) mod 11 = 48 mod 11 = 4 | T[4] |
| 5 | (8 + 50) mod 11 = 58 mod 11 = 3 | T[3] |
| 6 | (8 + 60) mod 11 = 68 mod 11 = 2 | T[2] |
| 7 | (8 + 70) mod 11 = 78 mod 11 = 1 | T[1] |
| 8 | (8 + 80) mod 11 = 88 mod 11 = 0 | T[0] |
| 9 | (8 + 90) mod 11 = 98 mod 11 = 10 | T[10] |
| 10| (8 + 100) mod 11 = 108 mod 11 = 9 | T[9] |

**Последовательность индексов:**
```
8 → 7 → 6 → 5 → 4 → 3 → 2 → 1 → 0 → 10 → 9
```

**Особенность:** Поскольку h₁(k) = 10 взаимно просто с m = 11 (НОД(10,11)=1), последовательность покрывает **все 11 ячеек таблицы** в обратном порядке (начиная с 8, затем убывание на 1 по модулю 11).

## Дополнительные вопросы для самопроверки:

### Вопрос 8: Объясните понятие "коллизия" в хеш-таблицах.

**Ответ:** Коллизия (столкновение) — это ситуация, когда два разных ключа после применения хеш-функции дают одинаковый индекс в хеш-таблице. Это неизбежно, так как количество возможных ключей U значительно превышает размер таблицы m. Существует два основных подхода к разрешению коллизий:
1. **Метод цепочек** — каждая ячейка содержит список элементов
2. **Открытая адресация** — поиск другой свободной ячейки

### Вопрос 9: Что такое коэффициент заполнения α и как он влияет на производительность?

**Ответ:** Коэффициент заполнения α = n/m, где n — количество элементов в таблице, m — размер таблицы. Чем больше α, тем больше коллизий и проб при зондировании. Рекомендуется поддерживать α ≤ 0.7 для обеспечения эффективной работы хеш-таблицы.

| α | Качество работы |
|---|-----------------|
| < 0.5 | Отличное |
| 0.5 - 0.7 | Хорошее |
| 0.7 - 0.85 | Удовлетворительное |
| > 0.85 | Плохое (много коллизий) |

### Вопрос 10: Сравните методы разрешения коллизий (линейное, квадратичное, двойное хеширование).

**Ответ:**

| Метод | Формула | Преимущества | Недостатки |
|-------|---------|--------------|-------------|
| Линейное | (h(k)+i) mod m | Простота, быстрота вычислений | Первичная кластеризация |
| Квадратичное | (h(k)+c₁i+c₂i²) mod m | Уменьшает кластеризацию | Может не покрыть все ячейки |
| Двойное хеширование | (h(k)+i×h₁(k)) mod m | Равномерное распределение, нет кластеризации | Дополнительные вычисления |
